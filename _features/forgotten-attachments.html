 forgotten-attachments Patch

Alert user when (s)he forgets to attach a file to an outgoing email.

1. Patch

To check if Mutt supports “ forgotten-attachments”, look for “ patch-forgotten-attachments”in the mutt version. See: Section 1.2, “Mutt Patches”.

Dependencies:

  • mutt-1.7.0

This patch is part of the NeoMutt Project.

2. Introduction

The 'forgotten attachments' patch provides a new setting for Mutt that alerts the user if the message body contains a certain keyword but there are no attachments added. This is meant to ensure that the user does not forget to attach a file after promising to do so in the mail.

3. Variables

Table 6.8. forgotten-attachments Variables

Name Type Default
attach-keyword string attached
abort_noattach quadoption ask-yes

4. Muttrc

# This is a complete list of forgotten-attachments-related configuration.
 
# --------------------------------------------------------------------------
# VARIABLES - shown with their default values
# --------------------------------------------------------------------------
 
# The 'forgotten attachments' patch provides a new setting for Mutt that
# alerts the user if the message body contains a certain keyword but there are
# no attachments added. This is meant to ensure that the user does not forget
# to attach a file after promising to do so in the mail.
 
# Search for the following keyword in the body of the email
set attach_keyword = "attached"
 
# Ask if the user wishes to abort sending if $attach_keyword is found in the
# body, but no attachments have been added
set abort_noattach = ask-yes
