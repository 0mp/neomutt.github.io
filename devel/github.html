GitHub Repositories

All the source code for NeoMutt, and its patches, is available on GitHub. The majority is in the neomutt repo.

Each feature has its own branch in the repo. Any changes to features happen there. This means each can be cleanly merged into Mutt.

The NeoMutt branch is mostly merge commits from the feature branches. It also contains commits that aren't appropriate to merge into Mutt, such as changing the version number.

Here are some diagrams showing alternative release strategies.
In the future, these release strategies may be adopted instead of the "release with hotfixes"

Periodic Release

After a period of time, all the features are merged into NeoMutt and a release is made.
The features are merged immediately before a release is made.

History

  • Two commits are made to the features
  • Features are merged into NeoMutt
  • Release R1 is made
  • More commits are made to the features.
  • Time passes (// on diagram)
  • Features are merged into NeoMutt
  • Release R2 is made

Advanced

This is the same as the hotfix method, but also incorporated the latest development features.

History

  • Two commits are made to the features
  • Features are merged into NeoMutt
  • Release R1 is made
  • Release is merged into Advanced A1
  • A commit is made to a feature
  • The commit is merged into NeoMutt
  • This becomes Hotfix H1
  • A commit is made to a another feature
  • The commit is merged into NeoMutt
  • This becomes Hotfix H2
  • Some rolling release distros may pick up those Hot Fixes
  • The hotfixes are merged into Advanced A2
  • A commit is made to a development feature
  • The commit is merged into Advanced A3
  • Release R2 is made
  • Release is merged into Advanced A4
